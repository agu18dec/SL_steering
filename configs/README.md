## Conventions

- **run_name** lives in the config itself; copy it verbatim into the row.
- **Hub artifact** — full HF Hub path (no prefix abbreviations).
- **Commit** — `git rev-parse --short HEAD` at the time of the run. Write "uncommitted" if the tree was dirty (expected; this branch never commits).
- **Notes** — anything that shouldn't be guessable from the config alone: upstream changes, sanity-check outcomes, deviations from the plan.

## Config files (one per pipeline stage)

Four files, every ablation/sweep rides as CLI overrides on top. See each file's docstring for invocation recipes.

- `gen.py` — canonical 30k teacher gen. Overrides: `use_system_prompt`, `temperature`, `run_name`, `size`.
- `filter.py` — two-stage filter (rule + Claude Haiku 4.5 judge; streaming stop at `target_size` NO). Overrides: `run_name`, `pilot_size`, `target_size`.
- `train.py` — LoRA SFT trainer. Overrides: `lora_r`, `lora_alpha`, `seed`, `run_name`.
- `eval.py` — animal-preference cat-rate eval on base or adapter. Overrides: `adapter_path`, `model`, `samples_per_prompt`.

**Do not create per-run config files.** Ablations go in the table above with the exact invocation line.
